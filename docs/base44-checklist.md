# צ'קליסט חיבור Base44 עם GitHub

## 🎯 מטרה
להבטיח חיבור יציב, נקי וצפוי בין GitHub ל-Base44 ללא איבוד קוד או בלגן.

---

## 1. הגדרת Git (מקור אמת)

- GitHub הוא מקור האמת (Source of Truth)
- branch ראשי: `main` (לא master)
- לא עובדים מחוץ לזרימה מסודרת
- לבצע commit לעיתים קרובות עם הודעות ברורות

---

## 2. מבנה פרויקט

- מבנה תיקיות ברור ועקבי
- ללא קבצי ניסוי או קבצים מיותרים
- קיים `README.md`
- תיקיית `docs/` לתיעוד
- קבצי config מרוכזים וברורים

---

## 3. משתני סביבה (Environment Variables)

- אין לשים secrets בתוך הקוד
- שימוש ב-`.env` לפיתוח מקומי
- `\.env.example` מכיל רק placeholders
- `.env` לא נכנס ל-Git (`.gitignore`)

---

## 4. יציבות מקומית (לפני Base44)

לפני חיבור ל-Base44:

- `npm install` עובד
- `npm run dev` רץ ללא שגיאות
- `npm run build` מצליח
- אין שגיאות import
- אין קריסות בזמן ריצה

---

## 5. הפרדה בין קוד לנתונים

- Git שומר קוד בלבד
- Base44 שומר נתונים (DB / Entities)
- מחיקת נתונים ב-Base44 ≠ מחיקת קוד ב-Git
- יש להתייחס לפעולות על DB כפעולות רגישות

---

## 6. הימנעות מ-AI Import (קריטי)

אם המטרה היא סנכרון Git אמיתי:

❌ לא להשתמש ב:
- Fetch דרך AI
- Writing (שחזור אוטומטי)
- יצירת אפליקציה מתוך URL

✅ כן להשתמש ב:
- חיבור GitHub
- Connect repository
- Git Pull

### ⚠️ אם בוצע AI Import בטעות:

- הפרויקט ב-GitHub לא נפגע
- לא נמחק קוד ולא השתנה דבר ב-Git
- Base44 יצר גרסה משוחזרת (לא clone אמיתי)

👉 מה עושים:
- מתעלמים או מוחקים את התוצאה
- מחברים מחדש את הריפו דרך Git
- מבצעים Pull נקי

---

## 7. חיבור נכון ל-GitHub ב-Base44

- לוודא שהריפו הנכון נבחר
- לא ליצור ריפו חדש אם כבר קיים
- אם הריפו לא מופיע → לבחור "All repositories"
- לוודא חשבון/ארגון נכון

---

## 8. זרימת עבודה מומלצת

### אופציה א' — עבודה דרך Claude Code / לוקאלי

1. שינוי קוד
2. `git add`
3. `git commit`
4. `git push` (ל-main)
5. Base44 → Git → Pull
6. בדיקה באפליקציה

---

### אופציה ב' — עבודה מתוך Base44

1. שינוי קוד בתוך Base44
2. Commit
3. Push ל-GitHub
4. GitHub משמש כגיבוי

---

## 9. פעולות הרסניות (זהירות!)

לפני אישור פעולות כמו:

- Delete
- Reset
- Clear
- Remove all

יש לבדוק:

- מה בדיוק נמחק?
- האם זה קוד או נתונים?
- האם יש גיבוי?
- האם זה באמת נדרש?

❗ לא לאשר פעולות כאלה בלי להבין

---

## 10. בדיקות אחרי Pull

לאחר כל Pull:

- הקבצים החשובים קיימים
- אין קבצים חסרים
- האפליקציה נטענת
- אין שגיאות console
- ה-UI עובד תקין

---
