# Chapter Engine — Root Cause Analysis Report

**Date:** 2026-06-22  
**Scope:** Full chapters subsystem — sources, display priority, boundary detection, title/summary generation, quality gates  
**Status:** Analysis only. No code was modified.

## 1. Current Chapter Sources

There are **7 distinct paths** that can produce or display chapters.

### 1.1 Description Timestamps (`description_timestamp`)
**File:** `src/services/youtubeMetadata.js:23–81`  
**Function:** `extractTimestampsFromDescription(description)`

Parses the YouTube video description for lines matching `\d{1,2}:\d{2}` pattern.  
Requires ≥2 valid matches; a single timestamp is ignored.  
The last chapter has `endSeconds = null` (end of video).

---

### 1.2 GEM Chapters (`gem`)
**File:** `src/components/dashboard/VideoDetailPanel.jsx:528–557`  
**Function:** `normalizeGemChapters(raw)`

Consumes `universalTabs.chapters` from a Gemini API GEM response.  
Accepts multiple time formats: numeric seconds, `"120"`, `"02:00"`, `"1:02:33"`.  
Falls back to `"פרק ${i+1}"` if title is missing.

---

### 1.3 AI Analysis Chapters (`gemini`, `transcript`, `saved`, `ai_generated`)
**File:** `src/services/videoAnalytics.js:1615–1674`  
**Function:** `resolveDisplayChaptersWithSource(video)`

Chapters previously generated by Claude or Gemini and stored on `video.aiChapters`.  
Already contain `startSeconds`, `endSeconds`, and pre-generated titles/summaries.

---

### 1.4 Topic Heuristic Chapters (`transcript_topic_heuristic`)
**File:** `src/lib/chapterTopicBoundaries.js:107–309`  
**Functions:** `detectTopicBoundaries()` → `buildChaptersFromBoundaries()` → `applyChapterQualityRules()`

Local transcript analysis using a sliding window scorer.  
Detects: transition phrases (+4), new macro theme (+3), new ticker symbols (+2).  
Returns `null` if fewer than 3 boundaries are found (forces fallback).

---

### 1.5 Equal-Chunk Heuristic Chapters (`transcript_heuristic`)
**File:** `src/lib/chapterTopicBoundaries.js:333–368`  
**Function:** `buildEqualChunkChapters(lines, video)`

Divides transcript lines into `min(6, max(4, round(lines.length / 40)))` equal groups.  
No topic detection — pure line-count division.

---

### 1.6 Plain Text Split Chapters (`manual_transcript`)
**File:** `src/components/dashboard/VideoDetailPanel.jsx:660–720`  
**Function:** `splitPlainTranscriptToChapters(text, videoDurationSeconds)`

Used when the transcript is a plain string (no line objects).  
Splits by paragraph boundaries and character targets (3–7 chunks based on text length).

---

### 1.7 Generic Outline / Estimated (`outline`, `estimated`)
**File:** `src/services/videoAnalytics.js:376–500`  
**Function:** `generateChapters(video, tags)`

Template-based chapters. Uses video category (Markets / AI / Dev / General) to pick a pre-written plan of 8–10 chapters.  
Assigns timestamps via `estimatedChapters()`: `startSeconds = Math.round(idx * durationSec / total)`.

## 2. Actual Display Priority

**File:** `src/components/dashboard/VideoDetailPanel.jsx:3314–3355`

```javascript
const displayChapters =
  descriptionChapters.length > 0
    ? descriptionChapters
    : gemChapters.length > 0
      ? gemChapters
      : aiAnalysisChapters.length > 0
        ? aiAnalysisChapters
        : transcriptChunkChapters.length > 0
          ? transcriptChunkChapters
          : baseChapters;
```

**Priority order (first non-empty wins):**

| Rank | Source | Type |
|------|--------|------|
| 1 | Description/YouTube timestamps | Real timestamps |
| 2 | GEM universalTabs.chapters | AI (Gemini) |
| 3 | Saved AI Analysis chapters | AI (cached) |
| 4 | Transcript Chunk chapters (topic OR equal) | Heuristic |
| 5 | Base chapters (generic outline) | Template |

### Critical finding: weak chapters can NOT override strong ones

Transcript-heuristic chapters (rank 4) are only shown if ranks 1–3 are all empty.  
This means: if a video has even one empty GEM chapter array that resolves to non-empty, or if `aiChapters` exists from an old poor-quality run, the transcript heuristic is completely suppressed — even if it would produce better results.

Conversely, if only a low-quality `outline` (rank 5) exists, the transcript heuristic will correctly override it.

**However:** there is no mechanism to rescue a video where stale AI Analysis chapters (rank 3) are worse than what the transcript heuristic (rank 4) would generate.

## 3. Boundary Generation Logic

| Source | How `startSeconds` is set | How `endSeconds` is set | Classification |
|--------|--------------------------|------------------------|----------------|
| Description timestamps | Parsed from `MM:SS` or `HH:MM:SS` in description text | Next chapter's `startSeconds` | **REAL / USER** |
| GEM chapters | From `ch.startSeconds` / `ch.start` / `ch.timestamp` | From `ch.endSeconds` / `ch.end` (if present) | **AI-GENERATED** |
| AI Analysis | Pre-stored on `video.aiChapters[].startSeconds` | Pre-stored `endSeconds` | **AI-GENERATED** |
| Topic Heuristic | Transcript line timestamp where score ≥ 3 | Next boundary's `startSeconds` or video duration | **TOPIC-DRIVEN (heuristic)** |
| Equal Chunk | Timestamp of first line in each equal block | Timestamp of first line of next block | **CHUNK-BASED** |
| Plain Text Split | `Math.floor((i / total) * durationSec)` | `Math.floor(((i+1) / total) * durationSec)` | **DURATION-BASED** |
| Generic Outline | `Math.round(idx * durationSec / total)` | Derived from next chapter's start | **DURATION-BASED** |

### The boundary quality mismatch

Paths 5, 6, and 7 all produce timestamps that are **mathematically derived from video duration**, not from actual content transitions. These timestamps appear plausible (they are real timecodes that exist in the video) but they do not correspond to any real topic change.

Path 4 (Topic Heuristic) does attempt content-based boundaries, but only for market-domain videos where its 10-theme macro library and ticker extraction can find signals. For general videos (earnings calls, political commentary, tutorials) it frequently falls back to Path 5.

## 4. Title Generation Logic

| Source | Title Method | Template/AI/Heuristic | Quality |
|--------|-------------|----------------------|---------|
| Description | Raw text after timestamp in description | USER INPUT | High |
| GEM | `ch.title \|\| ch.name \|\| ch.label` — Gemini API | AI-GENERATED | High |
| AI Analysis | Pre-stored title from previous Claude/Gemini run | AI-GENERATED (cached) | High |
| Topic Heuristic | Macro theme title OR ticker label OR fallback `"פרק N"` | HEURISTIC | Medium |
| Equal Chunk | First 8 words of segment, fillers stripped | TRANSCRIPT FRAGMENT | Low |
| Plain Text | Top 3 keywords by weight (ticker +4, digit +3, Hebrew +2, other +1) | KEYWORD EXTRACTION | Low |
| Generic Outline | Pre-written templates per category (Markets, AI, Dev, General) | TEMPLATE | Medium |

### Title generation for Topic Heuristic (detail)
**File:** `src/lib/chapterTopicBoundaries.js:191–219`

Priority within `buildTopicChapterTitle()`:
1. `boundary.signals.macroShift.titleHe` — e.g., `"מדיניות הפד ושינויי ריבית"`
2. Ticker list joined — e.g., `"ניתוח מניות AAPL ו-MSFT ומגמות שוק"`
3. Dominant macro theme detected in segment
4. Tickers detected in segment
5. Transition phrase detected — `"מעבר לנושא חדש בסקירת השוק"`
6. Hard fallback — `"פרק ${segmentIndex + 1} בנושא ${videoTitle.slice(0, 28)}"`

### Garbage filtering
**File:** `src/lib/chapterTitleUtils.js:47–60`  
`isObviousTranscriptFragmentTitle()` replaces titles that are:
- Empty
- `[music]`, `[applause]`, `[laughter]`, `[silence]`
- Leading with fillers: "and", "we", "so"

`sanitizeChaptersForDisplay()` applies this gate at render time and replaces garbage with `"חלק N"`.

**Gap:** The filter is very narrow. It does not catch arbitrary transcript phrases like `"you know what I think the problem is"` or numeric-only titles like `"2 3 4"`.

## 5. Summary Generation Logic

| Source | Summary Content | Generation Method |
|--------|----------------|------------------|
| Description | `''` (always empty) | None |
| GEM | `ch.summary \|\| ch.description \|\| ''` | Gemini AI text (may be 0–2 sentences) |
| AI Analysis | Pre-stored `description` field | Claude/Gemini AI text |
| Topic Heuristic | `segment.text.slice(0, 220)` | Raw transcript excerpt |
| Equal Chunk | `chunk.text.slice(0, 220)` | Raw transcript excerpt |
| Plain Text | First 2 lines of chunk | Raw text excerpt |
| Generic Outline | Pre-written static description per template chapter | Static template string |

### Summary quality gap

Only GEM and AI Analysis chapters have real AI-generated summaries. All transcript-generated paths produce raw transcript slices, which can be:
- Mid-sentence cuts
- Filler speech (`"so um like I was saying..."`)
- Repeated content from adjacent chapters (especially with equal-chunk splitting)

There is no post-processing step that improves or enriches these slices.

## 6. Root Cause

### The core problem: title quality vs. boundary quality mismatch

The system has two independent quality dimensions:

**Dimension A — Boundary quality**  
*Where* the chapter starts and ends. Is it a real topic transition?

**Dimension B — Title quality**  
*What* the chapter is called. Does the title describe the content?

The Title quality can be high even when Boundary quality is low.

### How the mismatch occurs

1. A video is analyzed. No description timestamps exist. GEM response has no chapters. `aiChapters` is empty.
2. `generateChaptersFromTranscript()` is called.
3. `tryTopicDrivenChapters()` finds only 2 boundaries (threshold requires ≥3). Returns `null`.
4. Fallback: `buildEqualChunkChapters()` divides the transcript into 5 equal blocks.
5. Block 3 starts at 12:40 (a random mid-sentence location in the middle of a discussion about Fed policy).
6. `buildTopicChapterTitle()` is called on block 3's text. It detects the pattern `\bfed\b` and assigns the title: `"מדיניות הפד ושינויי ריבית"`.
7. The user sees: `12:40 — מדיניות הפד ושינויי ריבית`.
8. The timestamp is wrong (12:40 is not a topic transition). The title is correct (Fed policy IS discussed). The display looks convincing.

**Result:** Smart-looking titles sitting on artificial timestamps. The user clicks a chapter, the video jumps to a mid-sentence location, and the experience feels broken.

### Secondary root cause: no quality source tagging in display

The UI does not differentiate between:
- Chapters with real timestamps (`timeSource: 'real'`)
- Chapters with AI-estimated timestamps (`timeSource: 'estimated'`)
- Chapters with heuristic-derived timestamps (`chapterSource: 'transcript_heuristic'`)

All three render identically. The user has no signal that a chapter timestamp is artificial.

### Tertiary root cause: stale AI Analysis chapters suppress heuristic

If a video has old, poor-quality `aiChapters` stored (from a previous analysis run), those chapters stay at priority rank 3 and permanently suppress the transcript heuristic (rank 4). There is no staleness check, no quality score comparison, and no user-facing indicator that the displayed chapters came from an old cached run.

## 7. Risk Assessment

### Safe to change

| Area | Risk | Reason |
|------|------|--------|
| UI: `timeSource` badge display | Low | Read-only, display-only change |
| `sanitizeChaptersForDisplay()` — widen garbage filter | Low | Replaces bad titles with neutral fallback |
| `buildEqualChunkChapters()` — add `timeSource: 'heuristic'` tag | Low | Adds a field, no schema breakage |
| `isQualityTopicTitle()` — relax 4-word minimum | Low | Only affects topic heuristic title filtering |
| Priority cascade — allow heuristic to override stale AI chapters | Medium | Changes display output; needs regression test |

### Risky to change

| Area | Risk | Reason |
|------|------|--------|
| `detectTopicBoundaries()` scoring algorithm | High | Affects all transcript-generated chapters; broad regression surface |
| GEM chapter normalization | High | GEM schema is external; defensive parsing must not break |
| `splitPlainTranscriptToChapters()` | Medium | Used as final fallback; breakage means zero chapters displayed |
| `generateChapters()` (generic outline) | Medium | Used as base for all new videos; changes affect first-run experience |
| `resolveDisplayChaptersWithSource()` priority logic | High | Core display logic; any error shows wrong or no chapters |

### Do not touch without explicit approval

- `extractTimestampsFromDescription()` — works correctly; real user data
- `normalizeGemChapters()` — GEM integration; break = no chapters from Gemini
- Any saved `aiChapters` field structure — stored in DB; schema change breaks existing videos

## 8. Recommended Fix Plan

### Phase 1 — Safe display priority cleanup (no content changes)

**Goal:** Show users what type of chapter boundary they are looking at.  
**Files:** `src/components/dashboard/ChapterItem.jsx`, `src/lib/chapterTitleUtils.js`  
**Risk:** Minimal — display-only

Changes:
- Add a `timeSource` badge to `ChapterItem` when `chapterSource` is `transcript_heuristic`, `transcript_topic_heuristic`, or `manual_transcript`
- Badge text: `"זמן משוער"` (estimated time) in amber color
- Badge text: `"זמן מדויק"` (exact time) in green for `description_timestamp`
- No changes to data, schemas, or priority logic

---

### Phase 2 — Mark weak chapters, prevent stale AI from suppressing heuristic

**Goal:** Stop old `aiChapters` from permanently overriding better heuristic output.  
**Files:** `src/services/videoAnalytics.js:1615–1674`, `src/components/dashboard/VideoDetailPanel.jsx:3314–3355`  
**Risk:** Medium — changes display priority

Changes:
- Add a `qualityScore` field to chapters at generation time
  - `description_timestamp` → 100
  - `gem`, `gemini`, `ai_generated` → 80
  - `transcript_topic_heuristic` → 60
  - `transcript_heuristic` → 30
  - `manual_transcript` → 25
  - `outline` / `estimated` → 10
- In priority cascade: if `aiChapters` exist but all have `qualityScore < 40` AND transcript heuristic produces chapters with score ≥ 60, prefer heuristic
- This prevents stale equal-chunk AI chapters from suppressing topic heuristic

---

### Phase 3 — Improve local topic boundary detection

**Goal:** Increase the rate at which `tryTopicDrivenChapters()` succeeds (currently falls through to equal chunks for non-market videos).  
**Files:** `src/lib/chapterTopicBoundaries.js`  
**Risk:** Medium — affects transcript chapter quality

Changes:
- Lower `MIN_BOUNDARY_SCORE` from 3 to 2 for videos where topic detection found at least 1 macro theme
- Add a generic "structure signals" detector: numbered lists (`"1.", "2."`, Hebrew `"א.", "ב."`), question words (`"מה", "איך", "why", "how"`), section dividers (`"---"`, repeated `"==="`), speaker changes in transcripts
- Add new transition patterns to `TRANSITION_PHRASE_RE`:  
  - `"לסכם"`, `"לסיום"`, `"בנוסף לכך"`, `"בהמשך"`, `"let me explain"`, `"the key point is"`
- Lower minimum boundary count from 3 to 2 (if video < 20 minutes)

---

### Phase 4 — AI topic segmentation with boundary justification

**Goal:** For high-value videos (macro analysis, earnings), trigger a focused AI call to identify real topic boundaries.  
**Files:** `src/services/aiVideoAnalyzer.js`, `src/components/dashboard/VideoDetailPanel.jsx`  
**Risk:** High — new AI call path; must be opt-in

Changes:
- Add `handleGenerateTopicChaptersWithAI()` as a **separate** button from the existing transcript button
- Send first 5,000 characters of transcript to Claude with prompt: identify timestamps where the topic genuinely shifts, return JSON array with `{startSeconds, title, reason}`
- Store `reason` field per chapter (can be shown as tooltip: "why this is a boundary")
- This is Phase 4 only — not part of the automatic pipeline

---

### Phase 5 — Validation and QA tools

**Goal:** Give the developer visibility into chapter quality without manual inspection.  
**Files:** New dev-only utility, `src/lib/chapterQualityDiagnostics.js`  
**Risk:** Minimal — dev tools only, not in production UI

Changes:
- `diagnoseChapters(chapters, transcript)` function:
  - Checks whether each `startSeconds` aligns within ±30 seconds of a transcript line
  - Checks whether each chapter title contains words that appear in the corresponding transcript segment
  - Returns a quality report: `{ alignment: 0.8, titleRelevance: 0.6, weakChapters: [2, 4] }`
- Console-only in development; no production output

## 9. QA Plan

For each video type below, define the **pass criteria** before testing in Base44 Production.

### 9.1 Macro/Market Analysis Video (e.g., Fed press conference summary)

**Setup:** Video with no YouTube description timestamps. Run "Generate Chapters from Transcript."

| Check | Pass | Fail |
|-------|------|------|
| Chapter count | 5–9 chapters | < 3 or > 12 |
| Chapter source | `transcript_topic_heuristic` | `transcript_heuristic` (equal chunk) |
| Timestamps | Each is within ±60 sec of a real topic shift | Timestamps are evenly distributed at equal intervals |
| Titles | Contain Fed, inflation, yields, or specific tickers | All titles start with "חלק N" |
| Summaries | Coherent excerpt from that segment | Mid-sentence cut-off in first word |

### 9.2 Technical Analysis Video (e.g., chart reading, indicators)

**Setup:** Video about technical setups (RSI, MACD, support/resistance).

| Check | Pass | Fail |
|-------|------|------|

| Ticker detection | $AAPL / $TSLA appear in chapter titles if mentioned | Generic "חלק N" throughout |
| Boundary type | Boundaries at pattern explanations, not mid-sentence | Boundaries at 1/5, 2/5, 3/5 intervals |
