לספק פרומפטים מוכנים לעבודה מסודרת על פרויקט Base44, בהתאם לכללים שב-AGENTS.md.

---

## Prompt בסיסי לעבודה על הפרויקט

הקשר:
אתה עובד על פרויקט Base44 עם GitHub כמקור אמת.

כללים מחייבים:
- לעבוד דרך קבצי הפרויקט בלבד
- לא להסתמך על DevTools Console אם אפשר לפתור דרך קוד, סקריפט או פונקציה
- לא להכניס secrets, tokens, API keys או סיסמאות לקוד
- לא לשנות קבצים שלא נדרשו
- לשמור placeholders בלבד בקבצי `.env.example`
- להניח ש-GitHub הוא מקור האמת של הקוד
- כשצריך לסנכרן ל-Base44, לעשות זאת דרך Git → Pull
- להסביר בקצרה מה שונה, אילו קבצים השתנו, ואיך לאמת את התוצאה

בקשה:
בצע את המשימה בצורה זהירה, עקבית ומתועדת.

## Prompt ליצירת פיצ'ר

הקשר:
אני רוצה להוסיף פיצ'ר חדש לפרויקט.

הנחיות:
- נתח קודם את הקבצים הרלוונטיים
- הצע שינוי מינימלי וברור
- בצע את השינוי רק בקבצים הנדרשים
- שמור על המבנה הקיים של הפרויקט
- אל תעביר לוגיקה חשובה לעבודה ידנית ב-Base44 אם אפשר לייצג אותה בקוד
- בסיום, פרט מה נוצר או עודכן ואיך לבדוק

בקשה:
הוסף את הפיצ'ר בהתאם למבנה הקיים של הפרויקט.

## Prompt לתיקון באג

הקשר:
יש באג בפרויקט שצריך לתקן בלי לשבור התנהגות קיימת.

הנחיות:
- אתר את מקור הבעיה לפני שינוי
- בצע תיקון ממוקד בלבד
- הימנע משינויים רחבים שלא נדרשים לפתרון
- אם יש סיכון לרגרסיה, ציין אותו
- בסיום, הסבר מה הייתה הסיבה לבאג ואיך התיקון פותר אותה

בקשה:
תקן את הבאג בצורה הכי מצומצמת ובטוחה שאפשר.

## Prompt לריוויו על שינוי קיים

הקשר:
צריך לבצע review לשינוי קיים בפרויקט.

הנחיות:
- התמקד קודם בבאגים, סיכונים, רגרסיות אפשריות וחוסרים
- תן עדיפות לממצאים עם השפעה ממשית
- אם אין ממצאים, אמור זאת במפורש
- אל תמליץ על Console אם אפשר לפתור דרך קבצים

בקשה:
בצע review ממוקד סיכונים, עם ממצאים ברורים והמלצה מעשית.

## Prompt לעבודה עם Environment Variables

הנחיות:
- לא להכניס ערכים אמיתיים לקוד או ל-Git
- אם צריך דוגמה, להשתמש רק ב-placeholders
- להבדיל בין ערכים לדוגמה לבין ערכים אמיתיים
- להניח שערכים אמיתיים יוגדרו ב-Base44 או בקובץ env מקומי שלא נכנס ל-Git

בקשה:
בצע את השינוי בלי לחשוף secrets ובלי להכניס ערכים אמיתיים לקבצים מנוהלים.

הקשר:
בוצעו שינויים בקוד המקומי וצריך להכין אותם לסנכרון מול Base44.

הנחיות:
- ודא שכל הלוגיקה החשובה מיוצגת בקבצים
- ודא שאין secrets בקוד
- ודא שהשינויים מתאימים לעבודה דרך GitHub
- תאר בקצרה את סדר הפעולות המומלץ: commit, push, ואז Base44 Git Pull
- אל תניח שיש סנכרון אוטומטי מ-GitHub

בקשה:
הכן את השינויים בצורה בטוחה ומסודרת לסנכרון מול Base44.

## ניסוח מוכן ל-Base44 אחרי commit + push

אם המשימה הושלמה ובוצע commit + push, החזר גם הודעה מוכנה להדבקה ב-Base44 בפורמט הבא:

### הודעה מוכנה ל-Base44
בוצע עדכון בקוד והועלה ל-GitHub.

פרטים:
- branch: [שם ה-branch]
- commit: [hash]
- push: בוצע

מה השתנה:
- [שינוי מרכזי 1]
- [שינוי מרכזי 2]
- [שינוי מרכזי 3]

הערות:
- השינויים הם בצד ה-frontend בלבד
- אין שינוי ב-secrets
- אין שינוי ב-environment variables

בבקשה לבצע:
Git → Pull → Publish
